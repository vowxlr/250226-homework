1. Поиск непрерывного подмассива с заданной суммой (для неотрицательных чисел).
..Алгоритм:
int summ = заданная сумма
int arr = { заданные числа } отсортированный !!
for int i = 1; i < arr.Leight;i++
    создаем список и добавляем arr[0] элемент
    if list.sum + arr[i] == summ
         мы нашли подмассив (подлист (?))
    else if list.sum + arr[i] < summ
         добавляем след элемент и считаем заново
    else if list.sum + arr[i] > summ
         очищаем список, добавляем туда след элемент (в нашем случае arr[1]) и считаем заново с i + 1 элемента
..Сложность
O(n) тк в худшем случае мы пройдёмся по всему массиву

2. Поиск "пика" в массиве (пик - элемент что не меньше своих соседей (достаточно одного соседа))
..Алгоритм:
будет подобие бинарного поиска
int arr = { заданные числа } неотсортированнный !!
left = 0, right = arr.Leight - 1
пока left < right:
    середина = (left + right) // 2
    if arr[середина] < arr[середина + 1]
           left = середина + 1
    else right = середина - 1
в конце концов в left будет пик
..Сложность
O(logn) тк в основе алгоритма бинарный поиск, сокращающий промежуток вдвое
